# api聚合反代 使用教程

## 1. 安装与启动

1. 下载 `安装包/api聚合反代-WIN-X64-1.0.20.exe`。
2. 双击安装包，根据安装向导完成安装。
3. 安装完成后打开 `api聚合反代`。
4. 应用启动后会在本机提供管理界面和 OpenAI 兼容接口。

默认管理界面：

```text
http://127.0.0.1:23456
```

默认 API 地址：

```text
http://127.0.0.1:23456/v1
```

## 2. 添加 API

可以通过两种方式添加 API。

### 手动添加

1. 打开 `API管理`。
2. 点击添加 API。
3. 填写接口名称、接口地址、API Key、模型名称、所属分组和价格信息。
4. 保存后可以进行状态测试。

### 模板导入

1. 打开 `模板导入`。
2. 下载空模板。
3. 按模板填写 API 信息。
4. 选择 `.xlsx` 文件导入。
5. 导入后到 `API管理` 检查分组与接口数量。

建议同一批接口按模型或用途分组，例如文本模型、图像模型、备用接口等。

## 3. 配置分组调用

分组名称会作为外部工具调用时的模型 ID。

例如有一个分组叫：

```text
gpt-5.5
```

外部工具里就可以把 `Model` 填成：

```text
gpt-5.5
```

应用会按照分组内 API 的顺序和状态进行转发。你可以在 `API管理` 中展开分组，拖拽 API 调整优先级。

## 4. 复制令牌并接入外部工具

1. 打开 `令牌管理`。
2. 复制默认令牌或新建一个令牌。
3. 在外部工具中填写：

```text
Base URL: http://127.0.0.1:23456/v1
API Key: 复制的令牌
Model: 分组名称
```

只要外部工具支持 OpenAI 兼容接口，一般都可以按这个方式接入。

## 5. 查看调用日志和用量

打开 `使用日志` 可以查看每次请求的状态、模型、Tokens、消耗、耗时和失败原因。

打开 `数据看板` 可以查看整体趋势、接口消耗、模型使用情况和成功率。这个页面适合用来判断哪些接口稳定、哪些接口成本较高、哪些接口需要调整优先级。

## 6. 查看网站资源分享

打开 `白嫖分享` 可以查看已经整理好的网站资源。普通用户页面会显示 `访问链接` 按钮，点击后会使用系统默认浏览器打开网页，不会在应用内部打开新窗口。

网站资源支持标签展示，适合按工具类型、使用场景或资源用途快速区分。发布者本人可以看到自己提交的完整链接，其他用户默认只看到访问按钮，页面更清爽，也能减少链接被随意复制传播。

## 7. 图像接口使用建议

如果某些 API 是图像接口，请在分组或 API 配置里按实际用途标记。图像接口和文本接口的测试方式不同，应用会根据类型使用更合适的测试请求。

图像接口常见场景：

- 文生图
- 图生图
- 商品图生成
- 图片编辑或变体生成

## 8. 本地数据与隐私

应用内的 API Key、接口地址、分组策略、调用日志等运行数据默认保存在本机。公开仓库只提供安装包和使用说明文档，不包含用户本地配置文件。

请勿主动公开以下文件：

```text
.env
database/
api.xlsx
本地导出的 API 配置文件
```

## 9. 常见问题

### 打不开管理界面

确认应用已经启动，并检查端口是否被占用。默认地址是：

```text
http://127.0.0.1:23456
```

### 外部工具提示连接失败

检查外部工具中的 `Base URL` 是否填写为：

```text
http://127.0.0.1:23456/v1
```

同时确认令牌没有复制错误。

### 某个 API 经常失败

可以在 `API管理` 中进行状态测试，也可以在 `使用日志` 中查看失败原因。如果某个接口不稳定，可以调整优先级或暂时停用。

### 导入后数量不对

先检查 Excel 模板中的分组名、接口地址、API Key 和模型名称是否填写完整。重复导入时，应用会优先更新已有 API，避免重复堆叠。

# 更新说明

## 1.0.34

发布日期：2026-07-14

- 默认移除内置云端 Supabase/Worker 地址，云端连接改为显式环境变量配置，避免分发安装包携带固定云数据库入口。
- 禁用用户 API Token 云端同步，用户导入的 API、Key、分组、日志和 token 继续仅保留在本地工作区。
- 云端 Worker 与 SQL 函数同步禁用敏感 token 写入，并对用户工作区快照中的 API、分组、日志和 token 做清理。
- 同步更新 Docker/NAS 部署子项目，并重新打包 Windows x64 安装包。

## 1.0.33

发布日期：2026-07-12

- 聊天页面支持回车发送，Shift+Enter 换行；页面整体固定自适应窗口，只允许聊天内容区域滚动。
- 聊天页面“指定分组内 API”改为常显下拉框，默认项由“自动选择”改为“按分组排序”，不再需要额外勾选。
- 应用管理页改为 Agent/Code 应用目录，不再展示 Web 控制台、OpenAI 接口等系统项目；已安装应用自动排序到前面。
- 优化应用检测准确率，增加开始菜单快捷方式、注册表卸载项、常见安装目录和 PATH 的综合检测。
- 已安装应用优先提取本机可执行文件原始图标展示，方便按真实应用图标区分。
- 修复登录账号与 API/分组/令牌/日志工作区绑定不稳的问题：登录后会认领本地无归属数据，云端为空时会推送本地工作区，云端和本地都有数据时合并，保持不同用户互相隔离。
- 数据看板继续按当前账号的 API、分组和日志统计，配合工作区同步后可跟随用户账号同步。
- 白嫖分享通知改为只对未读批次弹出；关闭或自动消失后，刷新页面或重新进入页面不会重复弹出同一批通知。
- 同步更新 Docker/NAS 部署子项目，并将发布仓库安装包目录清理为最多保留最新 3 个版本。
## 1.0.32

发布日期：2026-07-12

- 修复聊天窗口在历史消息或图片字段为空时可能出现 `Cannot read properties of null (reading 'image')` 的崩溃问题，并对本地对话记忆做兼容清洗。
- 优化聊天页面布局，左侧模型和配置选择栏保持固定，页面滚动只影响右侧对话内容。
- 修正重新发送、重新生成时复用错误请求 payload 的问题。
- 重做应用管理页面为应用启动器式布局，默认内置主流 Agent Code / 桌面 Agent 应用目录，支持自动检测、图标展示、已安装点亮、未安装置灰禁用和启动分组选择。
- 扩展应用目录，覆盖 Codex、Claude Desktop、Claude Code、Gemini Desktop、Gemini CLI、OpenCode Desktop、OpenCode、Trae、TRAE SOLO CN、Trae CN、Cursor、VS Code、Windsurf、Zed、ZCode、WorkBuddy、Hermes Desktop、Grok Build、Kiro、Void 等常见应用。
- 优化白嫖分享页面通知，改为右上角浮窗式消息栈，进入页面后自动展示并在数秒后收起，不再作为常驻面板占用页面空间。
- 同步更新 Docker/NAS 部署子项目，保持桌面主项目与 `docker部署api` 子项目功能一致。

## 1.0.31

发布日期：2026-07-12

- 应用管理页增强为主流 Agent Code 应用自动检测列表，已安装应用自动点亮并可启动，未安装应用保持暗色不可操作。
- 应用启动时可选择本次使用的模型分组，并通过环境变量注入默认模型分组，方便外部 Agent Code 应用直接调用对应分组。
- API 管理新增缓存读取价格、缓存创建价格两类按量计费字段，导入、编辑、保存和实际消耗计算均已支持。
- 聊天测试页支持对话记忆，切换页面后聊天记录不再重置；选择分组后可选定分组内单个 API 进行测试，默认仍为自动轮换。
- 数据看板“模型数据分析”中每个模型 ID 旁新增同字号复制按钮，可一键复制模型 ID，不改变原板块布局。
- 同步更新 Docker/NAS 副本代码，保持桌面主项目与 `docker部署api` 子项目功能一致。

## 1.0.30

发布日期：2026-07-10

- 新增“应用管理”页面，支持按账号维护常用应用列表、应用启动配置、代理注入设置和启动日志。
- 支持自动检测并添加 Web 控制台和 Visual Studio Code，VS Code 启动时可注入本项目代理地址环境变量。
- 支持为应用选择当前用户令牌，并通过 `API_AGGREGATOR_API_KEY` / `OPENAI_API_KEY` 注入，不修改系统全局配置、会话记录或技能仓库。
- 应用配置和启动日志纳入用户工作区，不同用户互不干扰。
- Docker/NAS 副本同步应用管理页面和配置能力，并默认禁用宿主机系统应用启动，避免多用户服务器环境误启动宿主机桌面应用。

## 1.0.29

发布日期：2026-07-10

- 修复浏览器、NAS、服务器和 Cloudflare 隧道网页模式下白嫖分享链接可能无法打开的问题。
- 白嫖分享链接点击时会先在用户浏览器中占位打开，拿到真实链接后跳转；如果新窗口被拦截，会回退到当前标签页打开。
- 扩展用户管理页面批量邮件目标，支持全部正常、7 天内到期、已到期、已停用、已封禁、注册申请邮箱和当前勾选用户。
- 注册申请窗口新增“给申请邮箱发邮件”入口，方便管理员通知申请人。
- 复核新用户默认无令牌、令牌按账号绑定、不同用户分组和工作区快照互不混入。
- 同步 Docker/NAS 部署副本，继续保持 Docker 默认端口 3039 与 Cloudflare 快速隧道配置。

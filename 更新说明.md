# 更新说明

## 1.0.36

发布日期：2026-07-14

- 新增并部署 Cloudflare Worker + D1 免费后端入口，公共安装包默认只内置 Worker 地址，不再内置数据库连接串、Supabase secret 或邮箱授权码。
- 桌面端云端配置改为本地优先安全策略：用户 API、分组、Token、调用日志、模型用量和工作区快照不上传云端。
- 修复 `DISABLE_CLOUD_BACKEND=1` 开关逻辑，现在会彻底禁用 Worker/Supabase 云端后端，避免私有开发环境中的旧变量被误用。
- Cloudflare Worker 侧继续阻断敏感同步 RPC，`user_tokens` 远端为空，普通表访问仅开放白嫖分享 `website_shares`。
- 同步 Docker/NAS 子项目，默认使用同一 Worker 入口，并补齐 Wrangler 部署依赖，方便一键部署/迁移。
- 重新打包 Windows x64 安装包，并完成敏感字符串扫描：安装包内未发现 Supabase URL、publishable key、secret key、数据库连接串、SMTP 授权码或相关 secret 标记。

## 1.0.35

发布日期：2026-07-14

- 切换并验证新的 Supabase 项目配置，本地开发环境支持新版 `SUPABASE_PUBLISHABLE_KEY` / `SUPABASE_SECRET_KEY` 变量名。
- 完成新数据库表结构、账号注册登录 RPC、用户管理 RPC、注册申请、白嫖分享与本地优先隐私加固迁移。
- 修复登录后工作区加载时 `claimOrphanRecords is not a function` 的兼容问题。
- 修复云端账号服务不可用或未配置时用户管理页加载失败的问题，自动回退本地用户列表。
- 继续禁止上传用户 API Key、分组、Token、调用日志、模型用量和工作区快照；云端仅保留账号资料和白嫖分享数据。
- 同步 Docker/NAS 子项目，支持新版 Supabase 环境变量、可选 Worker 地址和 SMTP 变量占位，便于一键部署。
- Windows 安装包继续排除 `.env`、`.env.*`、本地数据库、Excel/API 文件、日志、数据库连接串和邮箱授权码。

## 1.0.34 及更早版本

- 持续优化应用管理、聊天测试、API 管理、数据看板、用户隔离、Docker/NAS 部署和安装包中文显示。
- 历史版本安装包不再保留在发布仓库中，请优先使用最新版。
